九、少数用户启动 fatal error 的排障记录

用户反馈截图中的错误：

`FATAL ERROR: CAppSystemDict: Unable to create interface Source2ServerConfig001 from server (Dependency of application)`

用户复现条件：

1. 验证游戏完整性。
2. 重新安装插件。
3. 选择 BOT 模式。
4. 选择高难度。
5. 启动游戏，先出现正常全屏黑框。
6. 随后弹出该 fatal error，黑框消失，无法进入游戏。

初步判断：

这不是 `High/botprofile.vpk` 高难度文件本身的问题。

理由：

1. 高难度只影响 `overrides/High/botprofile.vpk`。
2. `botprofile.vpk` 不会创建 `Source2ServerConfig001` 这种 Source2 引擎接口。
3. 报错发生在游戏启动加载 server app system 阶段。
4. 这个阶段通常早于 C# 插件逻辑执行。

更合理的判断：

这是 BOT 模式启用后的本机底层加载链异常。

BOT 模式下 `gameinfo.gi` 会加入：

`Game csgo/addons/metamod`

这会让游戏加载：

`addons/metamod/bin/win64/server.dll`

如果该用户本机存在以下情况，就可能触发接口创建失败：

1. 旧版 Metamod 残留。
2. CounterStrikeSharp 残留。
3. `addons` 目录里有其他插件冲突。
4. 安装覆盖不完整。
5. 杀软或权限导致 DLL 复制失败。
6. Steam 验证完整性后没有清理第三方 `addons` 文件。
7. 使用特殊客户端、平台环境或非标准游戏分支。

由于该问题大约 200 人只有 1 人反馈，不应直接定性为整包普遍不兼容。

如果是整包里的 Metamod 普遍失效，应当大面积出现同样 fatal error。


十、建议给用户的处理流程

对出现该问题的用户，建议执行彻底清理，而不是只验证完整性：

1. 退出 CS2 和 Steam。
2. 打开 CS2 的 `game/csgo` 目录。
3. 删除 `gameinfo.gi`。
4. 删除 `addons/metamod`。
5. 删除 `addons/metamod.vdf`。
6. 删除 `addons/metamod_x64.vdf`。
7. 删除 `addons/counterstrikesharp`。
8. 删除 `overrides`。
9. Steam 验证游戏完整性。
10. 重新解压我们的整包。
11. 再选择 BOT 模式。

如果仍然复现，需要收集：

1. `game/csgo/gameinfo.gi`
2. `game/csgo/addons` 目录截图。
3. `addons/metamod/bin/win64/server.dll` 文件属性和版本。
4. `addons/metamod/bin/win64/metamod.2.cs2.dll` 文件属性和版本。
5. `addons/counterstrikesharp/bin/win64/counterstrikesharp.dll` 是否存在。
6. 用户是否装过其他 CS2 插件或平台工具。


十一、后续产品侧建议

建议后续安装器增加一个“修复安装 / 彻底重装 BOT 环境”按钮。

该功能执行：

1. 备份当前 `gameinfo.gi`。
2. 删除旧 `addons/metamod`。
3. 删除旧 `addons/counterstrikesharp`。
4. 删除旧 `overrides`。
5. 重新复制当前整包文件。
6. 重新写入 BOT 模式 `gameinfo.gi`。

这样能降低残留型问题概率。

另一个长期优化方向：

不要直接用发布包里的旧 `gameinfo.gi` 覆盖用户当前文件，而是在用户当前 CS2 版本的 `gameinfo.gi` 基础上插入必要 SearchPath。

这样可以降低 CS2 更新后，旧 `gameinfo.gi` 覆盖新文件导致启动阶段异常的风险。
